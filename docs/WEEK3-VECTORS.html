Week 3: Vector Search with Pinecone

What Are Embeddings?

Embeddings convert text into numbers that capture meaning.

Text Embedding (vector) ──── ────────────────── "cat" → [0.12, -0.34, 0.56, 0.78, ...] "kitten" → [0.11, -0.33, 0.57, 0.77, ...] ← Similar! "car" → [0.82, 0.15, -0.23, 0.44, ...] ← Different Similar meanings = vectors point in similar directions

How similarity works

We measure the angle between vectors using cosine similarity:

  • 1.0 = identical meaning
  • 0.0 = unrelated
  • -1.0 = opposite meaning
cosine_similarity("breathing problems", "dyspnea")           → 0.89
cosine_similarity("breathing problems", "respiratory distress") → 0.85
cosine_similarity("breathing problems", "headache")            → 0.23

OpenAI Embeddings API

import OpenAI from 'openai';

const openai = new OpenAI();

async function createEmbedding(text: string): Promise {
  const response = await openai.embeddings.create({
    model: 'text-embedding-3-small',  // 1536 dimensions
    input: text,
  });

  return response.data[0].embedding;
}

Model options

Model Dimensions Cost per 1M tokens Use case
text-embedding-3-small 1536 $0.02 Most use cases (we'll use this)
text-embedding-3-large 3072 $0.13 Higher accuracy needs
text-embedding-ada-002 1536 $0.10 Legacy (don't use)

Vector Databases

Why not just PostgreSQL?

You can store vectors in PostgreSQL (with pgvector), but:

  • Vector search is computationally expensive
  • Specialized indexes (HNSW, IVF) are needed for speed
  • Vector DBs are optimized for this one job

Pinecone Operations

1. Initialize client

import { Pinecone } from '@pinecone-database/pinecone';

const pinecone = new Pinecone({
  apiKey: process.env.PINECONE_API_KEY!,
});

const index = pinecone.Index('medical-notes');

2. Upsert vectors

await index.upsert([
  {
    id: 'doc-123-chunk-0',
    values: embedding,  // number[]
    metadata: {
      patientId: 'patient-abc',
      content: 'Patient reports chest pain...',  // Store text for retrieval
      date: '2024-01-15',
      sectionType: 'subjective',
    },
  },
]);

3. Query (search)

const queryEmbedding = await createEmbedding('breathing problems');

const results = await index.query({
  vector: queryEmbedding,
  topK: 5,
  includeMetadata: true,
  filter: {
    patientId: { $in: ['patient-abc', 'patient-xyz'] },  // Optional filter
  },
});

// results.matches = [
//   { id: 'doc-456-chunk-2', score: 0.89, metadata: {...} },
//   { id: 'doc-123-chunk-0', score: 0.85, metadata: {...} },
//   ...
// ]

Metadata Filtering

Pinecone can filter results before ranking by similarity.

// Only search notes from specific patients
filter: { patientId: { $in: ['p1', 'p2'] } }

// Only recent notes
filter: { date: { $gte: '2024-01-01' } }

// Only assessment sections
filter: { sectionType: { $eq: 'assessment' } }

// Combine filters
filter: {
  $and: [
    { patientId: { $eq: 'patient-abc' } },
    { sectionType: { $in: ['assessment', 'plan'] } },
  ]
}
Why this matters for hybrid RAG: We can first query PostgreSQL to find diabetic patients, then filter Pinecone to only search those patients' notes.

The Ingestion Pipeline

FHIR Documents │ ▼ ┌─────────────┐ │ Parse │ Extract DocumentReference resources └─────────────┘ │ ▼ ┌─────────────┐ │ Chunk │ Split into ~400 token pieces (Week 2) └─────────────┘ │ ▼ ┌─────────────┐ │ Embed │ OpenAI text-embedding-3-small └─────────────┘ │ ▼ ┌─────────────┐ │ Upsert │ Store in Pinecone with metadata └─────────────┘

Batch processing

// Don't embed one at a time—batch for efficiency
const BATCH_SIZE = 100;

for (let i = 0; i < chunks.length; i += BATCH_SIZE) {
  const batch = chunks.slice(i, i + BATCH_SIZE);

  // Embed batch
  const embeddings = await Promise.all(
    batch.map(chunk => createEmbedding(chunk.content))
  );

  // Upsert batch
  await index.upsert(
    batch.map((chunk, j) => ({
      id: chunk.id,
      values: embeddings[j],
      metadata: chunk.metadata,
    }))
  );
}
