Capstone Options

Choose one enhancement to implement. Each has a challenge file with detailed instructions.

Option 1: PII Obscuring

File: docs/CHALLENGE-PII.md

Add privacy protection to obscure patient names, dates, and other PII in query results.

  • Hash-based name pseudonymization
  • Date obscuring (keep year, hide day/month)
  • Regex patterns for SSN, phone, email

Skills: Regex, hashing, privacy engineering

Option 2: Reranking

File: docs/CHALLENGE-RERANK.md

Add Cohere reranking to improve search result quality.

  • Integrate Cohere Rerank API
  • Rerank vector search results
  • Compare before/after quality

Skills: API integration, search quality evaluation

Option 3: Conversation Memory

File: docs/CHALLENGE-MEMORY.md

Add conversation history so users can ask follow-up questions.

  • Store conversation context
  • Resolve pronouns ("What about his medications?")
  • Maintain patient focus across turns

Skills: State management, context handling

Option 4: Custom MCP Tool

File: docs/CHALLENGE-MCP-TOOL.md

Build a novel MCP tool that extends the system's capabilities.

  • Design tool interface and parameters
  • Implement the functionality
  • Document usage and demo

Skills: API design, MCP protocol

Option 5: Your Idea

Have a different idea? Pitch it! Possibilities:

  • Multi-modal (images, PDFs)
  • Additional data sources
  • Analytics dashboard
  • Export/reporting features

Requirement: Get instructor approval for scope

Requirements

Your capstone should include:

  1. Working code - Passes existing tests, doesn't break the app
  2. New tests - At least 3 test cases for your feature
  3. Documentation - Update README or add docs explaining your feature
  4. Demo - 3-5 minute demonstration of your feature
Use AI tools! This is encouraged. Show how you used Claude, Cursor, or other AI tools to help build your feature. Include interesting prompts or conversations in your demo.
