Week 4: Agents and Prompts

The Orchestration Problem

We have two retrieval systems. How do we know which to use?

User Query What's Needed
"What medications is John Smith taking?" SQL only (structured lookup)
"Notes about breathing problems" Vector only (semantic search)
"Diabetic patients with foot pain in their notes" Both! SQL → filter → Vector
"How many patients have hypertension?" SQL aggregation
Solution: Use an LLM to analyze the query and decide the strategy.

Query Analysis with Structured Outputs

We use OpenAI's structured outputs to get reliable JSON from the LLM.

The Zod Schema

import { z } from 'zod';

const QueryAnalysisSchema = z.object({
  intent: z.enum([
    'patient_lookup',      // Find specific patient
    'patient_search',      // Search patients by criteria
    'clinical_search',     // Search clinical notes
    'hybrid_search',       // Structured + semantic
    'analytics',           // Counts, aggregations
    'general'              // General medical question
  ]),

  requiresSQL: z.boolean(),
  requiresVector: z.boolean(),

  entities: z.object({
    patientName: z.string().optional(),
    conditions: z.array(z.string()).optional(),
    medications: z.array(z.string()).optional(),
    labTypes: z.array(z.string()).optional(),
  }),

  semanticQuery: z.string().optional(),  // For vector search
});

Using the Responses API

import { zodTextFormat } from 'openai/helpers/zod';

const response = await openai.responses.parse({
  model: 'gpt-4o-mini',
  input: [
    { role: 'system', content: ANALYSIS_PROMPT },
    { role: 'user', content: userQuery },
  ],
  text: {
    format: zodTextFormat(QueryAnalysisSchema, 'query_analysis'),
  },
});

const analysis = response.output_parsed;

The Analysis Prompt

const ANALYSIS_PROMPT = `You are a query analyzer for a medical records system.

Given a user query, determine:
1. The intent (what type of query is this?)
2. Whether we need SQL (structured data: patients, conditions, meds, labs)
3. Whether we need vector search (clinical notes, symptoms, observations)
4. Extract any entities mentioned (patient names, conditions, etc.)
5. If vector search is needed, what semantic query should we use?

Examples:
- "Find John Smith" → patient_lookup, SQL only
- "Notes about chest pain" → clinical_search, vector only
- "Diabetics with foot problems" → hybrid_search, SQL + vector
- "How many patients have diabetes?" → analytics, SQL only

Be precise. Extract exact entity names when mentioned.`;

Formatting Results for the LLM

The retrieved data becomes context for the response generation.

function formatResultsForLLM(result: QueryResult): string {
  const parts: string[] = [];

  if (result.sqlResults?.patients) {
    parts.push('## Matching Patients\n');
    for (const patient of result.sqlResults.patients) {
      parts.push(`### ${patient.name}`);
      parts.push(`- DOB: ${patient.birthDate}`);
      parts.push(`- Conditions: ${patient.conditions.join(', ')}`);
    }
  }

  if (result.vectorResults) {
    parts.push('## Relevant Clinical Notes\n');
    for (const note of result.vectorResults) {
      parts.push(`### ${note.patientName} - ${note.date}`);
      parts.push('```');
      parts.push(note.content);
      parts.push('```');
    }
  }

  return parts.join('\n');
}

The Response Generation Prompt

const SYSTEM_PROMPT = `You are a medical records assistant.

You help healthcare providers find patient information quickly and accurately.

Guidelines:
- Be concise and clinical in tone
- Always cite which patient/note the information comes from
- If information is not in the provided context, say so
- Never make up patient data
- Flag any concerning findings appropriately

When responding:
1. Directly answer the question
2. Provide relevant details from the records
3. Note any limitations (missing data, etc.)`;

Building the final prompt

const messages = [
  { role: 'system', content: SYSTEM_PROMPT },
  {
    role: 'user',
    content: `Context from medical records:

${formattedResults}

---

User question: ${userQuery}`
  }
];

Prompt Engineering Tips

Technique Example Why
Role setting "You are a medical records assistant" Sets appropriate tone and expertise
Explicit constraints "Never make up patient data" Prevents hallucination
Output format "Cite which patient the info comes from" Ensures traceability
Structured context Markdown headers, code blocks Helps LLM parse the data
Medical safety: Always remind the LLM it's a search tool, not a diagnostic system. Include disclaimers as appropriate.
