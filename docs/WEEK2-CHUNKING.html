Week 2: Chunking Documents

Why Chunking Matters

Before we can search clinical notes, we need to break them into smaller pieces.

The Problem: A clinical note might be 5,000 words. If we embed the whole thing as one vector, we lose precision—the embedding becomes a blurry average of everything in the note.

What chunking gives us:

  • Precision: Search returns the relevant paragraph, not a 10-page document
  • Context window: LLMs have token limits—chunks fit better
  • Cost: Smaller chunks = fewer tokens = lower embedding costs

Chunking Strategies

Strategy How it works Pros Cons
Fixed size Split every N characters Simple, predictable Cuts mid-sentence
Sentence Split on sentence boundaries Preserves meaning Uneven chunk sizes
Paragraph Split on double newlines Natural boundaries Paragraphs vary wildly
Semantic Use LLM to find topic boundaries Best quality Expensive, slow
Document-aware Use document structure (headers, sections) Respects format Requires parsing
For SOAP notes: We'll use document-aware chunking. SOAP notes have clear sections (S, O, A, P) that we should respect.

Chunk Size: The Tradeoff

Too Small (50 tokens) Too Large (2000 tokens) ───────────────────── ────────────────────── "Patient reports" Entire clinical note as one chunk ✗ No context ✗ Diluted embedding ✗ Many chunks to search ✗ Returns too much ✓ Very precise ✓ Full context Sweet Spot: 200-500 tokens ────────────────────────── A paragraph or section with enough context

The overlap question

Should chunks overlap? Example with 20% overlap:

Chunk 1: "Patient reports chest pain when climbing stairs. Pain is..."
Chunk 2: "Pain is sharp, 6/10, radiating to left arm. No shortness..."
Chunk 3: "No shortness of breath. Patient denies nausea or..."

Overlap helps: If a query spans a chunk boundary, overlap ensures we don't miss it.

Overlap costs: More chunks = more storage, more embedding cost, more search results to process.

SOAP Note Structure

Clinical notes follow the SOAP format:

S - Subjective
What the patient tells the doctor.
"I've had this cough for 2 weeks. It's worse at night."

O - Objective
What the doctor observes and measures.
"Vitals: BP 120/80, HR 72, Temp 98.6F. Lungs: wheezing bilateral."

A - Assessment
Doctor's diagnosis and clinical reasoning.
"Acute bronchitis, likely viral. Rule out pneumonia."

P - Plan
Treatment plan and next steps.
"Prescribe albuterol inhaler. Return if fever develops. Chest X-ray if no improvement in 5 days."
Our approach: Parse SOAP sections and chunk each section separately. Include section type in metadata so we can filter searches.

Token Counting

Embeddings are priced per token. We need to count tokens accurately.

import { encoding_for_model } from 'tiktoken';

const encoder = encoding_for_model('text-embedding-3-small');

function countTokens(text: string): number {
  return encoder.encode(text).length;
}

// Example:
countTokens("Patient reports chest pain")  // → 5 tokens
countTokens("The patient reports experiencing chest pain")  // → 7 tokens
Don't guess token counts! "500 characters ≈ 100 tokens" is only roughly true. Medical terms often tokenize unexpectedly.
